## 一、背景与问题分析

### 1.1 当前实现概述

当前文献筛选（`ScreeningAgent`）采用两阶段、多标准、确定性决策流水线：

```
Phase 0: LLM 生成筛选标准 → N 条 title 标准 + M 条 abstract 标准
Phase 1: 仅用 title 批量筛选 → 任何一条 NO → 直接 EXCLUDE
Phase 2: title+abstract 批量筛选 → 仅对 Phase 1 幸存者
Phase 3: 确定性合并决策 → 全 YES=INCLUDE, 有 NO=EXCLUDE
```

### 1.2 已识别的核心问题

| # | 问题 | 影响 |
|---|------|------|
| 1 | **标准生成质量不可控** — LLM 自由生成的 binary yes/no 问题质量参差不齐，且无验证机制 | 标准质量直接决定后续筛选上限 |
| 2 | **AND 决策规则过于严苛** — 所有标准全部 YES 才 INCLUDE | 大量应纳入的文献被错误排除，Sensitivity 低 |
| 3 | **Phase 1 早停风险大** — 仅凭 title 就用 NO 永久排除文献，title 信息量极度有限 | 不可逆的误排除 |
| 4 | **无 reasoning/chain-of-thought** — 筛选 prompt 直接要求 YES/NO，无推理过程 | LLM 判断质量低且不可审计 |
| 5 | **单次评估无纠错** — 每篇文献只评估一次，无复核机制 | 无法纠正随机错误 |
| 6 | **UNCERTAIN 处理模糊** — UNCERTAIN 既不纳入也不排除，最终不参与分析 | 可能丢失边界文献 |
| 7 | **无 publication type / study design 预筛选** — 未利用 PubMed 元数据 | 浪费 LLM 调用，噪声大 |

### 1.3 设计灵感来源

基于论文 *"Do it faster with PICOS: Generative AI-Assisted systematic review screening"*（Journal of Biomedical Informatics, 2025）的核心发现：

- LLM 生成结构化 PICOS 摘要辅助筛选，Sensitivity 达到 **100%**
- Accuracy 99.9%，Specificity 99.9%，F1 ~98%
- 筛选工作量减少 75%
- **核心理念：把"理解"和"判断"解耦** — 先让 LLM 提取结构化信息，再基于结构化信息做匹配判断

### 2.1 初始输入（文献检索阶段，系统界面入口）

## 四、各阶段详细设计

### 4.1 Stage 0: 规则预筛选

#### 4.1.1 目标

零 LLM 成本快速排除明显不相关文献，减少后续 LLM 调用量。

#### 4.1.2 排除规则

**硬排除列表（所有情况下都排除的 publication_type）：**

```python
EXCLUDED_PUB_TYPES = {
    # 综述/指南类
    "Review", "Systematic Review", "Meta-Analysis",
    "Guideline", "Practice Guideline",
    # 评论/通信类
    "Editorial", "Letter", "Comment",
    # 报告/新闻类
    "Case Reports", "News",
    # 其他非原始研究
    "Biography", "Published Erratum",
    "Retracted Publication", "Retraction of Publication",
}
```

**研究设计分类：**

```python
RCT_PUB_TYPES = {
    "Randomized Controlled Trial",
    "Clinical Trial",
    "Clinical Trial, Phase I",
    "Clinical Trial, Phase II",
    "Clinical Trial, Phase III",
    "Clinical Trial, Phase IV",
    "Controlled Clinical Trial",
    "Pragmatic Clinical Trial",
    "Equivalence Trial",
}

OBSERVATIONAL_PUB_TYPES = {
    "Observational Study",
    "Cohort Study",
    "Case-Control Study",
    "Cross-Sectional Study",
    "Comparative Study",
    "Multicenter Study",
    "Twin Study",
    "Validation Study",
    "Longitudinal Study",
}
```

#### 4.1.3 过滤逻辑

```python
def stage0_filter(paper: Paper, study_design_filter: StudyDesignFilter) -> str:
    """
    返回值：
      "KEEP"                    — 保留，进入 Stage 1
      "EXCLUDED_pub_type"       — 被硬排除列表排除
      "EXCLUDED_study_design"   — 被研究设计过滤器排除
    """
    pub_types = parse_publication_types(paper.publication_type)
    # pub_types 是一个 set，因为一篇文献可能有多个 publication_type

    # Step 1: 硬排除
    if pub_types & EXCLUDED_PUB_TYPES:
        return "EXCLUDED_pub_type"

    # Step 2: 研究设计过滤
    if study_design_filter == StudyDesignFilter.BOTH:
        return "KEEP"

    is_rct = bool(pub_types & RCT_PUB_TYPES)
    is_obs = bool(pub_types & OBSERVATIONAL_PUB_TYPES)

    if study_design_filter == StudyDesignFilter.RCT_ONLY:
        if is_obs and not is_rct:
            return "EXCLUDED_study_design"
        # is_rct=True 或 两者都不是（未知）→ KEEP
        return "KEEP"

    if study_design_filter == StudyDesignFilter.OBSERVATIONAL_ONLY:
        if is_rct and not is_obs:
            return "EXCLUDED_study_design"
        return "KEEP"

    return "KEEP"
```

**核心原则：信息不足时一律保留，交给后续 LLM 阶段判断。**

#### 4.2.4 Study Design 交叉验证

Stage 1 提取的 `S_study_design` 与用户设置的 `study_design_filter` 做二次验证：

```python
def stage1_study_design_check(
    extracted_design: str,
    study_design_filter: StudyDesignFilter,
) -> bool:
    """
    返回 True = 保留, False = 排除。
    仅在 Stage 0 因 publication_type 为空而放行的文献上执行。
    """
    if study_design_filter == StudyDesignFilter.BOTH:
        return True

    RCT_DESIGNS = {"RCT", "Quasi-experimental"}
    OBS_DESIGNS = {"Cohort", "Case-control", "Cross-sectional", "Before-after"}

    if extracted_design in {"Not reported", "Other", "Mixed methods"}:
        return True  # 不确定 → 保留

    if study_design_filter == StudyDesignFilter.RCT_ONLY:
        return extracted_design in RCT_DESIGNS or extracted_design not in OBS_DESIGNS

    if study_design_filter == StudyDesignFilter.OBSERVATIONAL_ONLY:
        return extracted_design in OBS_DESIGNS or extracted_design not in RCT_DESIGNS

    return True
```

#### 4.2.5 执行方式

- 使用 `batch_function_call_llm` 批量并发处理
- 模型：`qwen/qwen3.6-plus`（默认筛选模型）
- 并发度：`max_concurrency`（默认 50）
- temperature: 0.0（确保提取一致性）

#### 4.3.4 子步骤 2c — 决策规则（高召回率导向）

LLM 返回 `overall_decision` 后，系统用确定性规则做最终校验（防止 LLM 判断偏差）：

```python
def decide_v2(dimensions: dict) -> str:
    """
    高召回率决策规则。

    核心原则：
    1. 只有 P 或 I 明确 MISMATCH 才排除（这是系统综述最核心的两个维度）
    2. C/O/S 的 MISMATCH 不直接导致排除，降级为 UNCERTAIN
    3. 信息不足（UNCERTAIN）倾向保留
    """
    p = dimensions.get("P", "UNCERTAIN")
    i = dimensions.get("I", "UNCERTAIN")
    c = dimensions.get("C", "UNCERTAIN")
    o = dimensions.get("O", "UNCERTAIN")
    s = dimensions.get("S", "UNCERTAIN")

    # -------- 硬排除：P 或 I 明确不匹配 --------
    if p == "MISMATCH" or i == "MISMATCH":
        return "EXCLUDE"

    # -------- 强纳入：P+I 均匹配，且无任何 MISMATCH --------
    all_dims = [p, i, c, o, s]
    if p == "MATCH" and i == "MATCH" and "MISMATCH" not in all_dims:
        return "INCLUDE"

    # -------- 其他情况 → UNCERTAIN（交给 Stage 3 复核）--------
    # 包括：
    #   - P 或 I 为 UNCERTAIN（信息不足）
    #   - P+I 匹配但 C/O/S 有 MISMATCH
    return "UNCERTAIN"
```

**决策规则对照表：**

| P | I | C/O/S | 决策 | 说明 |
|---|---|-------|------|------|
| MATCH | MATCH | 全部 MATCH 或 UNCERTAIN | **INCLUDE** | 最理想情况 |
| MATCH | MATCH | 有 MISMATCH | **UNCERTAIN** | P+I 对但其他维度有问题，需复核 |
| MATCH | UNCERTAIN | 任意 | **UNCERTAIN** | I 信息不足，需复核 |
| UNCERTAIN | MATCH | 任意 | **UNCERTAIN** | P 信息不足，需复核 |
| UNCERTAIN | UNCERTAIN | 任意 | **UNCERTAIN** | 信息严重不足，需复核 |
| MISMATCH | 任意 | 任意 | **EXCLUDE** | P 明确不匹配 |
| 任意 | MISMATCH | 任意 | **EXCLUDE** | I 明确不匹配 |

### 4.4 Stage 3: 边界文献复核

#### 4.4.1 目标

对 Stage 2 判定为 UNCERTAIN 的文献进行更审慎的二次评审，给出最终 INCLUDE/EXCLUDE 决定。

#### 4.4.2 触发条件

- Stage 2 的 `overall_decision == "UNCERTAIN"` 的所有文献

#### 4.4.3 输入

复核时整合前两阶段的全部证据：

1. **Stage 1 输出**：该文献的结构化 PICOS 摘要
2. **Stage 2 输出**：逐维度推理链 + 各维度判定结果 + 哪些维度导致了 UNCERTAIN
3. **原始信息**：title + abstract
4. **Review 上下文**：完整 PICO 定义 + 匹配标准
5. **可选 — 用户上传的全文 PDF**：如果用户在复核阶段为该文献上传了 PDF

#### 4.4.4 全文 PDF 处理逻辑

```python
def prepare_fulltext_context(pdf_path: str, pico: PICODefinition) -> str:
    """
    当用户为某篇 UNCERTAIN 文献上传了全文 PDF 时，
    解析并提取与 PICO 相关的关键段落。

    复用现有的 pdf_parser.py + chunker.py 模块。
    """
    # Step 1: 用 Docling 解析 PDF → markdown
    parsed = parse_pdfs([pdf_path])
    markdown_text = parsed[0].text
    tables = parsed[0].tables

    # Step 2: chunking
    body_chunks, table_chunks = chunk_document(markdown_text, tables)

    # Step 3: BM25 检索与 PICO 相关的 top-k 段落
    pico_queries = [
        ("Population", pico.P),
        ("Intervention", pico.I),
        ("Comparison", pico.C),
        ("Outcome", pico.O),
    ]
    relevant_chunks = build_context_chunks(
        body_chunks, table_chunks,
        field_tuples=pico_queries,
        top_k=10,
    )

    return format_chunks_with_citations(relevant_chunks)
```

### 7.2 run() 方法流程

```python
def run(self, papers, pico, study_design_filter=StudyDesignFilter.BOTH, max_concurrency=50):
    self.reset()

    # Stage 0: 规则预筛选
    kept_papers, stage0_excluded = self._run_step(
        "stage0_filter", self._stage0_filter, papers, study_design_filter
    )

    # Stage 1: PICOS 提取
    picos_list = self._run_step(
        "stage1_extract_picos", self._stage1_extract_picos, kept_papers, max_concurrency
    )

    # Stage 1 附加: study_design 交叉验证
    stage1_kept, stage1_excluded = self._run_step(
        "stage1_design_check", self._stage1_design_check, kept_papers, picos_list, study_design_filter
    )

    # Stage 2a: 生成匹配标准
    criteria = self._run_step(
        "stage2_generate_criteria", self._stage2_generate_criteria, pico, study_design_filter
    )

    # Stage 2b: 逐维度匹配
    dimension_results = self._run_step(
        "stage2_match", self._stage2_match, stage1_kept, picos_filtered, criteria, max_concurrency
    )

    # Stage 2c: 应用决策规则
    decisions = self._merge_all_decisions(
        papers, stage0_excluded, stage1_excluded,
        stage1_kept, picos_filtered, dimension_results
    )

    # 汇总
    summary = self._compute_summary(decisions)
    return ScreeningResultV2(criteria=criteria, decisions=decisions, summary=summary)
```
